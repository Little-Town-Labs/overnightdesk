# Hermes Titus Runtime Contract

## Phase paths

| Path | Required keys | Runtime use |
| --- | --- | --- |
| `/agents/hermes-titus/runtime` | `OPENROUTER_API_KEY`, `AGENTMAIL_API_KEY`, `AGENTMAIL_INBOX_ID`, `AGENTMAIL_EMAIL_ADDRESS`, `HERMES_DEFAULT_MODEL` | Core model, dedicated email MCP/inbox, and default routing |
| `/agents/hermes-titus/overnightdesk` | `CONTROL_TOWER_TOKEN` | Control Tower caller authentication |
| `/agents/hermes-titus/teams` | `TEAMS_CLIENT_ID`, `TEAMS_CLIENT_SECRET`, `TEAMS_TENANT_ID`, `TEAMS_ALLOWED_USERS`, `TEAMS_ALLOW_ALL_USERS`, `TEAMS_PORT`, `TEAMS_HOME_CHANNEL`, `TEAMS_HOME_CHANNEL_NAME`, `TEAMS_DELIVERY_MODE`, `TEAMS_TEAM_ID`, `TEAMS_CHANNEL_ID` | Optional TTS Microsoft Teams activation |

The loader MUST reject unexpected keys, malformed exports, unexpected filesystem types, and core values that are empty or `NOT_CONFIGURED`. It MUST omit all Teams variables from the container process when any required Teams value is not ready. `TEAMS_ALLOW_ALL_USERS` MUST equal `false` in production.

## Container contract

- Name: `hermes-titus`
- Image: `overnightdesk/hermes-agent:0.18.0-coder`
- User: `10000:10000`
- Volume: `hermes-titus-data:/opt/data`
- Secret mount: `/run/hermes-titus/runtime.env:/run/secrets/hermes-titus-runtime:ro`
- Network: `overnightdesk_overnightdesk`
- Restart owner: systemd service, not an independent Docker restart policy
- Security: `--cap-drop=ALL`, `--security-opt=no-new-privileges`, 1 CPU, 2 GiB memory
- Published ports: none while Teams is pending

## Internal health contract

- Hermes dashboard: `http://127.0.0.1:9119/api/status`
- TencentDB memory gateway: `http://127.0.0.1:8420/health`
- Future Teams webhook health: `http://127.0.0.1:3978/health`
- Control Tower session: `http://control-tower:8080/v1/session` with the runtime bearer token

Health evidence MUST show only status, identifiers already returned by Control Tower, and key-presence booleans. It MUST NOT output bearer tokens, model keys, email keys, Teams secrets, or authorization headers.

## Memory contract

- Package version is pinned to 0.3.6.
- Provider directory name is exactly `memory_tencentdb`.
- Gateway binds to `127.0.0.1:8420`.
- Data stays below `/opt/data/memory-tencentdb/data`.
- The provider exposes memory and conversation search through Hermes.
- No memory port is published or attached to the shared Docker network.

## AgentMail contract

- The MCP endpoint is `https://mcp.agentmail.to/mcp`.
- The `x-api-key` header uses `${AGENTMAIL_API_KEY}` interpolation from the
  Phase-loaded process environment; no credential literal is stored in YAML.
- Titus identifies his dedicated inbox before mailbox work and does not use
  another agent's inbox.
- Read and summary requests are non-mutating.
- Sends, replies, forwards, draft changes, inbox changes, and other mailbox
  mutations require explicit operator approval immediately before execution.

## Teams activation contract

Teams activation is a separate production change. It requires:

1. TTS tenant values generated by the Microsoft Teams CLI or Azure registration.
2. An explicit `TEAMS_ALLOWED_USERS` list of AAD object IDs.
3. `TEAMS_ALLOW_ALL_USERS=false`.
4. A valid public TLS endpoint ending in `/api/messages`.
5. A successful local health check and authenticated Teams message smoke test.
6. An updated production deployment record.
